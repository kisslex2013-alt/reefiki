# source-command-new

Use this skill when the user asks to run the migrated source command `new`.

## Command Template

# /new <имя>

Аргумент: имя проекта в kebab-case (например, `crypto`, `gamedev`, `cooking`).

## Шаги

1. **Проверки.**
   - Если `projects/<имя>/` уже существует → сообщи и остановись.
   - Если cwd ≠ корень `REEFIKI/` → сообщи «эта команда работает только из корня» и остановись.

2. **Скопировать шаблон:** скопируй `projects/_template/` → `projects/<имя>/` (используй команду копирования текущей оболочки).

3. **Заменить `NAME` и `PROJECT_NAME`** во всех файлах нового проекта: `AGENTS.md`, `CLAUDE.md`, `_domain.md`, `wiki/index.md`, `wiki/log.md`, `wiki/_dashboard.md`.

4. **Интерактивное заполнение `_domain.md`.** Задай пользователю **ровно три вопроса** (не больше, не меньше), по-человечески, без терминов:

   - «О чём этот проект в одном-двух предложениях?»
   - «Откуда обычно будут источники? (статьи, видео, репозитории, PDF, твой опыт…)»
   - «Что в этой теме для тебя важнее всего — на что обращать внимание?»

   Получи ответы. Заполни `projects/<имя>/_domain.md` по шаблону: тема, типичные источники, ключевые entities (выведи из ответов), словарь (если упомянул синонимы), что искать особенно внимательно.

5. **Инициализировать `wiki/index.md`** — оставить заголовок и пустые секции (`## Sources`, `## Entities`, `## Concepts`, `## Synthesis`, `## Decisions`, `## Skills`). Без записей.

6. **Записать в `wiki/log.md`:**

   ```markdown
   ## [YYYY-MM-DD] meta | project initialized
   - Создано через /new из корня REEFIKI.
   - Домен: <одна фраза из ответа пользователя>.
   ```

7. **Git-коммит** (если репозиторий инициализирован): `git add -A ; git commit -m "new project: <имя>"`.

8. **Финал.** Сказать пользователю по-человечески:

   > Готово. Проект `<имя>` создан в `projects/<имя>/`. Открой папку проекта в любом IDE — агент подхватит правила из `AGENTS.md`. Первый шаг — скажи «сохрани <ссылка>» чтобы положить что-то в копилку.

## Запреты

- Не пытайся работать с проектом из корня после создания.
- Не заполняй `_domain.md` без ответов пользователя — он должен сам сформулировать. Если молчит — оставь плейсхолдер и помечь tag `triage`.
- Не создавай первые страницы в `wiki/` — это делается через `/process`/`/harvest` уже из проекта.
