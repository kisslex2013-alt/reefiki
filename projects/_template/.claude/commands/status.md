# /status — состояние проекта

> Эта команда только читает.

## Шаги

Быстрый путь: выполнить из корня репо `python scripts/reefiki.py --project projects/<name> status`, затем при необходимости дополнить вывод человеком.

1. **Inbox.** Посчитай файлы в `inbox/`. Если >0 — собери имена.
2. **Seen / quarantine.** В каждом `seen/*.md` прочитай `quarantine_until`. Посчитай: сколько истекли (≤ сегодня), сколько активны.
3. **Wiki.** Посчитай страницы по типам в `wiki/index.md` (sources, entities, concepts, synthesis, decisions, skills). Найди `use_count: 0` старше 60 дней (stale).
4. **Лог.** Найди последнюю запись `/lint`. Посчитай сколько `/process` было после неё.
5. **Health.** Если пользователь спрашивает о качестве/рисках базы или перед cleanup/phase switch — выполнить `python scripts/reefiki.py --project projects/<name> health`.
6. **Inbox-возраст.** Найди дату самого старого файла в `inbox/`.
7. **Индекс поиска.** Проверить наличие `.reefiki/index.sqlite`. Если нет — сказать, что поиск пересоберётся при первом `/query`.
7. **Планы.** Если есть `plans/*.md`, показать активные планы и напомнить про recovery.

## Вывод (по-человечески)

```text
Состояние проекта <NAME>:

Копилка: 3 файла (самый старый — 5 дней назад)
   • github-com-anthropic-mcp.md
   • karpathy-podcast-2025.txt
   • some-blog-post.md

Отказы: 14 (1 истёк срок проверки — посмотри в /lint)

Вики: 28 страниц (5 sources, 8 entities, 10 concepts, 3 synthesis, 2 decisions, 0 skills)
   • 5 страниц ни разу не использовались за 60+ дней

Последняя проверка здоровья: 14 дней назад / 7 /process с тех пор
Локальный поиск: индекс есть
Планы: 1 активный

Что предлагаю сейчас:
→ Разобрать копилку: /process
→ Через 3 /process будет пора /lint
```

Если **всё чисто** (inbox пуст, нет истёкших quarantine, /lint свежий):

```text
Состояние проекта <NAME>: всё в порядке.
Вики: 28 страниц. Копилка пуста. Последний /lint 3 дня назад.
```

## Запреты

- Не пиши никаких файлов.
- Не делай git commit.
- Не предлагай команды, которые не следуют из реального состояния.
