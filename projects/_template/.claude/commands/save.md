# /save — capture в inbox

> Эта операция самодостаточна. Анализа нет — только capture.

Аргумент: URL или путь к файлу. Никакого анализа на этом шаге. Только capture.

## Safety-фильтр (до записи в `inbox/`)

Быстрый путь: из корня репо выполнить `python scripts/reefiki.py --project projects/<name> privacy` для текущего проекта. Для конкретного нового файла/URL применить правила ниже до записи.

Откажись сохранять и скажи пользователю по-человечески, что не возьмёшь, если файл/путь:

- Имя/маска: `.env`, `.env.*`, `secrets.*`, `credentials.*`, `id_rsa`, `id_rsa.pub`, `*.pem`, `*.key`, `*.pfx`, `*.p12`, `.npmrc`, `.netrc`, `.aws/credentials`.
- Папка-источник: `.git/`, `node_modules/`, `vendor/`, `__pycache__/`, `dist/`, `build/`, `target/`.
- Размер: > 5 MB.
- Бинарник: расширение из `exe dll so dylib bin iso img dmg zip tar gz 7z rar mp4 mov mkv mp3 wav jpg jpeg png gif pdf docx xlsx pptx` без явной просьбы пользователя сохранить именно его. PDF/docx — спроси: «Это бинарник. Сначала извлеки текст и положи как `.md`/`.txt` в `inbox/`. Сохранить как есть всё равно?».

URL-вариант: если URL ведёт на бинарник по расширению (`.zip`, `.exe`, `.iso` и т.п.) — отказ с тем же сообщением.

Сообщение об отказе: «Не сохраняю `<имя>` — это похоже на `<секрет/бинарник/большой_файл>`. Если нужно именно его — переименуй в `inbox/` руками или дай явное согласие».

## Проверка дубликатов (до записи в `inbox/`)

Перед созданием файла в `inbox/` проверь, не обрабатывался ли этот источник ранее:

- Быстрый путь: из корня репо выполнить `python scripts/reefiki.py --project projects/<name> dedup "<URL или путь>"`.
- **URL-дубликат:** команда ищет тот же URL в `wiki/`, `inbox/` и `seen/`. Если найден — скажи: «Уже есть: `<существующая страница или inbox-файл>`. Сохранить ещё раз?» и жди подтверждения.
- **Файл-дубликат:** команда проверяет совпадение имени в `inbox/` и exact content hash (`sha256`) против файлов в `wiki/`, `inbox/` и `seen/`. Если найден — предупреди и жди подтверждения.

Если пользователь явно подтверждает — сохраняй с суффиксом `-2` к slug или под исходным именем, не перезаписывая.

## Шаги

1. **URL** → создать `inbox/<slug>.md` с одной строкой — этот URL.
   `<slug>` = домен + хвост пути, lowercase, через дефисы (например `github-com-karpathy-llm-wiki.md`).
2. **Путь к файлу** → скопировать файл в `inbox/` (имя оставить или нормализовать в kebab-case).
3. **Append в `wiki/log.md`**: `## [YYYY-MM-DD] /save | <source>`.
4. **Ответ пользователю**: `Сохранено в inbox. Записей в inbox: <N>. Запусти /process когда соберёшь батч.`

## Запреты

- Не читать содержимое URL.
- Не создавать страницы в `wiki/`.
- Не делать git commit (батчуется в `/process`).
