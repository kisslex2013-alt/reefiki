# Проблема авто-компакта и развитие REEFIKI

---

## Проблема: Почему авто-компакт контекста — это мусор

При активной работе с AI-агентами окно контекста рано или поздно заполняется. Редакторы предлагают «умное» решение — авто-компакт: модель сжимает историю, выделяет «главное» и едет дальше. На практике это ломает логику работы.

### Три корневые причины

**1. Нюансы важнее общего смысла**
Саммаризация сохраняет канву задачи, но выбрасывает конкретные названия переменных, хитрые костыли и — самое критичное — причины, по которым вы отказались от определённых архитектурных решений. Именно эти детали определяют качество кода.

**2. Эффект испорченного телефона**
Сжатый и искажённый контекст становится новой базой. Модель начинает галлюцинировать, восстанавливая утерянные куски из «воображения», а не из реальной кодовой базы. Каждый следующий компакт усугубляет искажение.

**3. Размытие фокуса**
Авто-старт тащит за собой хвост старых проблем и давно закрытых багов. Модель продолжает учитывать контекст того, что уже не требует её внимания, — и теряет фокус на текущей задаче.

---

## Текущее решение: Правило 70–80% (ручной подход)

Золотое правило: при заполнении окна на 70–80% — берёте управление на себя.

**Алгоритм:**
1. Останавливаетесь на 70–80% лимита
2. Открываете чистую сессию
3. Пишете жёсткий хэндофф-промпт вручную: актуальный сниппет + что сделано + что нужно + стек + ограничения

Занимает одну минуту, но требует дисциплины и держится на вас, а не на системе.

---

## Как REEFIKI решает это сейчас

REEFIKI переводит ручной подход в системный: вместо хаотичного авто-компакта — управляемая база знаний вне контекста.

| Проблема | Решение в REEFIKI |
|---|---|
| Потеря архитектурных решений | `wiki/decisions/` — постоянное хранилище с причинами выбора |
| Ручной хэндофф | `/harvest` фиксирует выводы, `/query` достаёт нужное в новой сессии |
| Размытие фокуса | Знания живут вне контекста и подключаются точечно, не целиком |
| Потеря навыков и паттернов | `wiki/skills/` — библиотека проверенных решений |

## Предложения по развитию

### 1. Команда `/checkpoint` — явный триггер перехода

Сейчас `/harvest` сохраняет выводы, но не готовит переход. Нужна команда, которая делает всё за один шаг:

- фиксирует выводы текущей сессии в вики
- генерирует готовый хэндофф-промпт
- кладёт его в `inbox/` для немедленного использования

Открыл новый чат → вставил промпт из `inbox/` → продолжил с нуля без потерь.

---

### 2. Авто-триггер по сигналам деградации контекста

Не ждать 70–80%, а замечать признаки раньше. Агент видит паттерны деградации:

- переспрашивает то, что уже обсуждалось
- предлагает решения, которые уже были отвергнуты
- начинает ходить по кругу

Эти паттерны прописываются в `AGENTS.md` как триггеры для предложения `/checkpoint`. Агент сам скажет: «Похоже, контекст перегружен — зафиксировать выводы и начать чисто?»

---

### 3. Обязательное поле `rejected_alternatives` в `decisions/`

Сейчас решения сохраняются, но причина отказа от альтернатив — не всегда. А именно это auto-compact выбрасывает первым.

**Структура записи:**
```
## Решение: [название]
- Выбрано: [что]
- Почему: [причина]
- Rejected alternatives:
  - Redux — слишком тяжёл для нашего стейта
  - MobX — команда не знает, порог входа высокий
- Контекст: [стек, дата]
```

Следующая сессия не переизобретает велосипед — она читает причины.

---

### 4. `context-budget.md` — живая карта сессии

Файл, который агент ведёт внутри сессии и обновляет по ходу работы:

```markdown
## Текущая задача
[что делаем прямо сейчас]

## Уже решено в этой сессии
- [X] реализована авторизация через JWT
- [X] отказались от cookie-based (причина: ...)

## Активные ограничения
- не трогать auth-модуль
- структура БД заморожена

## Следующий шаг
[что будем делать]
```

При смене сессии этот файл становится основой хэндофф-промпта автоматически — не надо вспоминать, что было сделано.

## Итог: от реактивного к проактивному

| | Авто-компакт | Ручное правило 70–80% | REEFIKI сейчас | REEFIKI с развитием |
|---|---|---|---|---|
| Сохраняет нюансы | ✗ | Частично | ✓ | ✓ |
| Требует дисциплины | Нет | Высокая | Средняя | Низкая |
| Проактивен | Да (плохо) | Нет | Нет | Да (хорошо) |
| Работает между сессиями | ✗ | ✗ | ✓ | ✓ |
| Генерирует хэндофф | Авто (мусор) | Вручную | Частично | Авто (качественно) |

Ключевой сдвиг: от «агент сам что-то сжал» к «я сам решаю, что сохранить, а агент помогает это сделать быстро и структурированно».
