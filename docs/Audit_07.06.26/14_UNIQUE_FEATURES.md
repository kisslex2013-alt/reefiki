#### Уникальные функции — реальные и логичные

Вот что вытекает из уже существующей архитектуры и философии, не выдумано:

**1. Contradiction Detector (автоматический)**
Сейчас `## Conflicting claims` добавляется агентом вручную. Можно автоматически при `/process` сравнивать новую страницу с существующими через FTS5 — если находит похожие страницы с противоречащими утверждениями, предлагает секцию конфликта. Это прямое продолжение существующего механизма, просто автоматизированное.

**2. Session Replay**
При старте новой сессии агент сейчас читает `memory pack`. Можно добавить `reefiki replay` — краткое резюме последней сессии: что сохранили, что решили, что осталось незакрытым. Данные уже есть в `wiki/log.md` и `plans/`. Просто нет команды которая их красиво агрегирует.

**3. Knowledge Decay / Confidence Aging**
Страницы стареют. Решение принятое год назад может быть неактуальным. Добавить поле `review_after: YYYY-MM-DD` в frontmatter и команду `reefiki review-due` — показывает страницы у которых истёк срок пересмотра. Это не геймификация, это реальная проблема устаревания знаний.

**4. Cross-Project Query**
Сейчас `/query` работает только внутри одного проекта. Но часто знание из проекта "python" нужно в проекте "work". Добавить `reefiki search --all-projects "asyncio"` — поиск по всем проектам сразу с указанием источника. Данные уже есть, просто нет глобального поиска.

**5. Provenance Chain**
Сейчас у страницы есть `sources` — список URL. Но нет связи "эта страница была создана на основе вот этой сессии, вот этого решения". Добавить `derived_from: [slug]` в frontmatter — тогда можно строить цепочку "почему я в это верю". Уже упоминалось в ERRATA, но не реализовано.

**6. Export / Publish отдельной страницы**
Сейчас publish — это весь public snapshot. Но часто нужно поделиться одной страницей — отправить коллеге, опубликовать в блог. `reefiki export --page python-async-patterns --format md/html/pdf`. Данные уже в markdown, просто нет команды экспорта.

**7. Inbox Triage Mode**
Когда в inbox накопилось 20+ элементов — обрабатывать их по одному неудобно. Добавить `reefiki triage` — режим быстрого просмотра: показывает элемент, ждёт ответа `s` (save), `d` (discard), `l` (later). Как `git add -p` но для inbox. В дашборде это уже частично есть, но в CLI нет.

**8. Template Gallery**
Сейчас один шаблон `_template`. Но разные домены требуют разных схем — research wiki отличается от engineering wiki. Добавить `reefiki new --template research|engineering|personal` — набор готовых шаблонов с разными наборами типов страниц и `_domain.md`. Это не новая архитектура, просто несколько вариантов существующего шаблона.

---

#### Что точно НЕ стоит добавлять

Чтобы не потерять фокус:

- **Векторный поиск** — FTS5 достаточно до 10K страниц, векторный поиск добавляет зависимость и сложность без измеримой пользы сейчас
- **Автоматическое создание страниц без участия пользователя** — нарушает философию "дистилляция требует суждения"
- **Синхронизация между пользователями** — это другой продукт (team tier), не core
- **AI-суммаризация при сохранении** — `/save` намеренно дешёвый, анализ — в `/process`
