## 3.1 Профиль типичной wiki-страницы

```
Средний размер страницы: 2–5 KB (500–1500 слов)
Среднее количество чанков на страницу: 3–8 (по заголовкам)
Среднее количество ссылок на страницу: 2–5 [[wikilinks]]
Среднее количество тегов: 3–6
```

## 3.2 Оценки производительности `build_index()`

| Страниц | Чанков | Время (SSD) | Время (HDD) | Размер sqlite |
|---|---|---|---|---|
| 100 | ~500 | < 0.5 сек | ~1 сек | ~2 MB |
| 500 | ~2500 | ~2 сек | ~5 сек | ~10 MB |
| 1000 | ~5000 | ~4 сек | ~12 сек | ~20 MB |
| 5000 | ~25000 | ~20 сек | ~60 сек | ~100 MB |
| 10000 | ~50000 | ~45 сек | ~150 сек | ~200 MB |

**Узкое место:** `DROP TABLE + CREATE TABLE` в начале каждого `build_index()` — полный rebuild при каждом `/reindex`. При 1000 страниц и HDD — 12 секунд блокирующей операции.

## 3.6 Рекомендации — немедленно (без рефакторинга)

```python
# 1. Увеличить cache_size SQLite (по умолчанию 2MB → 64MB)
conn.execute("PRAGMA cache_size = -65536")  # 64MB

# 2. Включить memory-mapped I/O
conn.execute("PRAGMA mmap_size = 268435456")  # 256MB

# 3. Использовать executemany вместо цикла execute
conn.executemany(
    "INSERT INTO pages VALUES (?,?,?,?,?,?,?,?,?)",
    rows_batch
)
```

## 3.7 Инкрементальный индекс (среднесрочно)

```python
def build_index_incremental(project: Path) -> int:
    db = db_path(project)
    if not db.exists():
        return build_index_full(project)

    db_mtime = db.stat().st_mtime
    changed = [p for p in iter_pages(project)
               if p.stat().st_mtime > db_mtime]

    if not changed:
        return 0  # индекс актуален

    conn = sqlite3.connect(db)
    for page in changed:
        _update_page_in_index(conn, page)
    conn.commit()
    return len(changed)
```
