# REEFIKI — Аудит: Лицензирование и юридика

> Дата: 4 июня 2026. Часть комплексного аудита незатронутых тем.

## 1.1 Текущее состояние

В репозитории **нет файла `LICENSE`**. По умолчанию применяется закрытое авторское право: никто не вправе использовать, копировать, модифицировать или распространять код без явного разрешения автора. Формально проект не является open-source, даже если он публично доступен на GitHub/GitLab.

## 1.2 Юридические последствия отсутствия лицензии

| Сценарий | Последствие |
|---|---|
| Пользователь форкает репо | Технически нарушает авторское право |
| Компания использует код внутри | Юридический риск без явного разрешения |
| Контрибьютор делает PR | Его вклад автоматически под закрытым копирайтом автора |
| Автор хочет монетизировать | Нет правовой базы для dual-licensing |
| Автор хочет привлечь инвесторов | Неопределённость IP отпугивает |

## 1.3 Анализ вариантов лицензии

**MIT / Apache 2.0 — максимальный adoption:**
- Разрешает использование, модификацию, коммерческое применение
- Apache 2.0 дополнительно защищает от патентных претензий
- Риск: конкурент может поднять managed-сервис на вашем коде без ограничений
- Подходит если цель — максимальное распространение и community

**BSL (Business Source License) — open-core защита:**
- Код открыт, но коммерческое использование ограничено N лет
- Через N лет автоматически переходит в MIT/Apache
- Используют: MariaDB, HashiCorp Vault, Sentry
- Подходит если планируется managed-сервис (REEFIKI Cloud)

**AGPL-3.0 — copyleft для SaaS-защиты:**
- Любой, кто использует код в сетевом сервисе, обязан открыть свой код
- Эффективно блокирует конкурентов от создания закрытого SaaS
- Риск: корпорации избегают AGPL-зависимостей, adoption ниже

**Dual-licensing (MIT + коммерческая):**
- Open-source версия под MIT, коммерческое использование по платной лицензии
- Сложнее администрировать, требует CLA от контрибьюторов

## 1.4 Рекомендация

Исходя из заявленной стратегии монетизации (open-core, Sync/backup как managed-сервис):

```
Рекомендация: Apache 2.0 для core + BSL для premium-компонентов

Структура:
- scripts/reefiki.py, reefiki_memory/, tests/ → Apache 2.0
- scripts/reefiki_sync/ (будущий Sync-сервис) → BSL 1.1
- Коммерческие коннекторы → проприетарная лицензия
```

## 1.5 CLA (Contributor License Agreement)

При dual-licensing или BSL **обязателен CLA**. Без него автор не может перелицензировать код, содержащий чужие PR.

Минимальный CLA — файл `CLA.md` + чекбокс в шаблоне PR:

```markdown
<!-- .github/pull_request_template.md -->
- [ ] Я прочитал CLA.md и согласен с его условиями
```

## 1.6 Авторские права на контент вики

Wiki-страницы, созданные пользователем — его интеллектуальная собственность. Зафиксировать в README:

```markdown
## Авторские права
Код REEFIKI: © 2026 [Автор], лицензия Apache 2.0.
Контент wiki-страниц: принадлежит пользователю, создавшему их.
REEFIKI не претендует на права на знания, хранимые в вашей вики.
```
